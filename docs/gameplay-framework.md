# 2D 牧羊合作游戏 — 玩法框架文档

> **验证命题**: 2D 牧羊合作游戏 = 羊群 AI 流动 + 空间地形控制 + 多人围控协作

---

## 一、品类调研摘要

### 1.1 核心参考作品

| 作品 | 年份 | 核心机制 | 关键启示 |
|------|------|---------|---------|
| **Sheepherds!** | 2025 | 牧羊犬 + 吠叫驱赶 + 无压力合作 | 羊群跟随行为、零惩罚设计、合作无分工 |
| **FLOCK!** | 2009 | UFO 驱赶 + 环境谜题 + 动物差异 | 不同动物不同反应、地形机关、关卡编辑器 |
| **Sheep (绵羊传奇)** | 2000 | 牧羊人/犬 + 危险规避 + 计分 | 角色多样性、时间+存活计分 |
| **Kyon** | 2017 | 犬 + 领头羊(Horatio) + 间接引导 | "领头羊"可视化重心、降低认知负担 |
| **Sheepy** | 学术 | Boids 驱赶 + 限时围栏 | 纯 Boids 算法验证 |

### 1.2 品类共性提炼

所有成功的"赶羊类"游戏都包含三个不可缺少的系统层：

```
┌───────────────────────────────────────────────┐
│        玩家操作层 (Player Agency)              │
│   移动 · 吠叫/惊吓 · 道具 · 协作站位          │
├───────────────────────────────────────────────┤
│        羊群行为层 (Flock Behavior)             │
│   Boids 群聚 · 逃离反应 · 个体差异 · 领头羊    │
├───────────────────────────────────────────────┤
│        环境约束层 (Environment)                │
│   地形阻挡 · 通道瓶颈 · 危险区域 · 目标围栏    │
└───────────────────────────────────────────────┘
```

**核心乐趣来源**：玩家不直接控制羊，而是通过**施加"排斥力"**间接影响羊群流向——这种"间接控制"产生的不确定性和涌现行为是品类的核心体验。

## 二、系统结构

### 2.1 系统架构总览

```
┌─────────────────────────────────────────────────────────┐
│                     Game Director                        │
│              (关卡加载 · 胜负判定 · 难度调控)              │
├──────────┬──────────┬──────────────┬────────────────────┤
│  Player  │  Flock   │ Environment  │   Cooperation      │
│  System  │  AI      │ System       │   System           │
│          │          │              │                    │
│ · 移动   │ · Boids  │ · 地形网格   │ · 合作评分         │
│ · 吠叫   │ · 逃离   │ · 障碍/围栏  │ · 站位引导         │
│ · 冲刺   │ · 状态机 │ · 危险区域   │ · 信号系统         │
│ · 道具   │ · 领头羊 │ · 通道/瓶颈  │ · 角色互补         │
└──────────┴──────────┴──────────────┴────────────────────┘
```

### 2.2 六大核心系统

#### S1: 玩家系统 (Player System)

```
玩家角色: 牧羊犬
├── 基础移动
│   ├── 八方向移动 (速度 > 羊的正常速度)
│   └── 冲刺 (短暂加速, 有冷却)
├── 驱赶技能
│   ├── 吠叫 (Bark)
│   │   ├── 范围: 以犬为圆心的扇形区域
│   │   ├── 效果: 对范围内的羊施加强排斥力
│   │   └── 冷却: 1.5s
│   └── 存在威压 (Presence)
│       ├── 范围: 犬周围的恒定圆形区域
│       └── 效果: 羊自然远离犬 (弱排斥力)
└── 信号系统
    ├── 快捷标记 (在地图上放置"来这里"标记)
    └── 表情 (简易沟通: 催促/等待/求助)
```

#### S2: 羊群 AI 系统 (Flock AI) — 核心系统

```
每只羊 = 独立 Agent, 行为由力的叠加决定
├── Boids 基础力 (群聚行为)
│   ├── Cohesion  (聚合): 靠近邻近同伴
│   ├── Separation(分离): 避免挤压重叠
│   └── Alignment (对齐): 朝向与邻居趋同
├── 环境交互力
│   ├── Flee      (逃离): 远离犬/威胁源
│   ├── Obstacle  (避障): 绕开地形障碍
│   └── Boundary  (边界): 远离地图边缘
├── 个体状态机
│   ├── Idle      (闲逛): 缓慢随机漫步
│   ├── Grazing   (吃草): 停留在草地上
│   ├── Alarmed   (警觉): 抬头观察, 准备逃跑
│   ├── Fleeing   (逃跑): 高速远离威胁
│   └── Panicked  (恐慌): 失控乱跑 (多次连续惊吓)
└── 特殊角色
    ├── 领头羊 (Leader): 影响力更大, 其他羊更倾向跟随
    └── 倔强羊 (Stubborn): 逃离反应弱, 需要更近距离驱赶
```

#### S3: 环境系统 (Environment System)

```
2D 俯视角关卡地图
├── 地形类型
│   ├── 草地 (通常): 正常移动
│   ├── 泥地 (减速): 犬和羊都减速
│   ├── 水域 (阻挡): 不可通行, 需绕道
│   ├── 灌木 (半阻挡): 犬可穿过, 羊不行
│   └── 围栏 (刚性阻挡): 都不可通过
├── 功能物件
│   ├── 栅栏门 (可开关): 需要玩家交互打开/关闭
│   ├── 目标围栏 (终点): 羊进入后计数
│   ├── 食物诱饵 (吸引): 放置后吸引附近的羊
│   └── 桥梁 (通道): 窄通道, 羊需排队通过
└── 危险区域
    ├── 狼出没区: 羊进入后会恐慌散开
    └── 悬崖边缘: 羊跑出边界则丢失
```

#### S4: 合作系统 (Cooperation System)

```
多人围控协作 (2-4人)
├── 围控机制
│   ├── 包围角度奖励: 多犬从不同方向包围 → 羊群更稳定
│   ├── 驱赶效率: 正确站位 → 羊群朝目标方向流动
│   └── 失控惩罚: 所有犬同侧 → 羊从另一侧逃散
├── 角色分工 (自然涌现, 非强制)
│   ├── 前锋: 在目标方向相反侧驱赶
│   ├── 侧翼: 在两侧防止羊群扩散
│   └── 收尾: 跟在羊群后面处理掉队的
├── 协作工具
│   ├── 小地图: 显示所有犬和羊的位置
│   ├── 快捷标记: 标记"推向这里"
│   └── 自动语音提示: "右边有羊跑了!"
└── 协作评价
    ├── 围控覆盖率: 犬的空间分布是否有效
    ├── 羊群紧凑度: 羊群是否保持整体
    └── 路径效率: 实际路径 vs 最短路径
```

#### S5: 关卡系统 (Level System)

```
关卡结构
├── 胜利条件: 将 N 只羊赶入目标围栏
├── 星级评价
│   ├── ★: 完成基本目标 (≥60% 的羊进栏)
│   ├── ★★: 良好完成 (≥80% + 较快速度)
│   └── ★★★: 完美完成 (100% + 高效路径 + 无羊丢失)
├── 难度曲线
│   ├── 第1关: 3只羊, 开阔场地, 直线路径 → 教学
│   ├── 第2关: 5只羊, 引入围栏门 → 学习开关门
│   ├── 第3关: 8只羊, L形路径 → 学习转向驱赶
│   ├── 第5关: 12只羊, 分散起始 → 学习先聚拢再驱赶
│   ├── 第8关: 15只羊 + 倔强羊 → 学习个体差异处理
│   └── 第12关: 20只羊 + 分叉路 + 狼区 → 综合挑战
└── 关卡变体
    ├── 聚集型: 羊散落各处, 先聚再赶
    ├── 护送型: 长距离路线, 维持队形
    ├── 分流型: 不同颜色的羊赶到不同围栏
    └── 防守型: 保护羊群免受狼的袭击
```

#### S6: 游戏导演系统 (Game Director)

```
动态难度调控
├── 监测指标
│   ├── 玩家完成进度 (已入栏 / 总数)
│   ├── 羊群紧凑度 (标准差)
│   ├── 玩家挫败感 (反复失败次数)
│   └── 平均操作频率 (APM)
├── 调控手段
│   ├── 羊群逃离速度 ± 调节
│   ├── 领头羊出现概率提升
│   ├── 环境提示 (箭头引导)
│   └── 额外食物诱饵刷新
└── 设计理念: 参考 Sheepherds! 的"无压力"哲学
    ├── 无时间限制 (基础模式)
    ├── 无失败惩罚 (羊丢失可重试该只)
    └── 每次操作都是正向贡献
```

## 三、玩法循环

### 3.1 核心循环 (30秒)

```
        ┌──────────────┐
        │   观察羊群    │ ← 判断: 羊在哪? 朝哪跑?
        │   位置分布    │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │   规划站位    │ ← 决策: 我该去哪? 从哪边赶?
        │   选择路线    │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │   移动到位    │ ← 执行: 跑到目标位置
        │   逼近羊群    │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │   施加驱赶    │ ← 行动: 吠叫/逼近, 羊群开始移动
        │   (吠叫/逼近) │
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │   观察反馈    │ ← 评估: 羊群流向对不对?
        │   调整策略    │         有没有羊掉队?
        └──────┬───────┘
               │
               └──→ 回到"观察羊群" (循环)
```

### 3.2 关卡循环 (3-8分钟)

```
开局 → 侦察羊群分布
  ↓
阶段1: 聚拢 (Gather)
  │  将散落的羊赶到一起形成羊群
  │  关键策略: 从外围向中心驱赶
  ↓
阶段2: 驱赶 (Drive)
  │  将羊群整体推向目标方向
  │  关键策略: 多人包围, 控制流向
  ↓
阶段3: 入栏 (Pen)
  │  引导羊群通过栅栏门进入围栏
  │  关键策略: 瓶颈控制, 防止回流
  ↓
结算 → 星级评价 → 解锁装饰/下一关
```

### 3.3 会话循环 (20-40分钟)

```
选择关卡 (或随机挑战)
  ↓
匹配/邀请伙伴 (1-4人)
  ↓
[关卡循环 × 3~5关]
  ↓
查看统计 (最佳合作搭档、效率排行)
  ↓
解锁奖励 (犬皮肤、帽子、表情包)
  ↓
继续/退出
```

## 四、羊群 AI 算法模型

### 4.1 Boids 力学模型

每只羊的运动由多个力矢量叠加决定：

```
最终速度 = 当前速度 + Σ(各力 × 权重) × dt

力的组成:
┌────────────────────────────────────────────────────┐
│  F_total = w_c · F_cohesion                        │
│          + w_s · F_separation                      │
│          + w_a · F_alignment                       │
│          + w_f · F_flee                             │
│          + w_o · F_obstacle                         │
│          + w_b · F_boundary                         │
│          + w_l · F_leader_follow                    │
│          + w_g · F_graze_attraction                 │
└────────────────────────────────────────────────────┘
```

#### 各力的计算方式

```
[Cohesion 聚合力]
  方向 = (邻居平均位置 - 自身位置).normalized
  强度 = w_c (默认 1.0)
  邻居半径 = R_cohesion (默认 5.0 米)

[Separation 分离力]
  对每个过近的邻居:
    方向 = (自身位置 - 邻居位置).normalized
    强度 = w_s / distance² (距离越近力越大)
  最小距离 = R_separation (默认 0.8 米)

[Alignment 对齐力]
  方向 = 邻居平均速度方向
  强度 = w_a (默认 0.8)
  邻居半径 = R_alignment (默认 3.0 米)

[Flee 逃离力] ← 核心驱赶力
  对每个威胁源(犬):
    方向 = (自身位置 - 犬位置).normalized
    强度 = w_f / distance  (距离越近力越大)
    衰减 = 超过 R_flee 则为 0
  吠叫加成: 犬吠叫时 w_f 临时 ×3, R_flee 临时 ×1.5

[Obstacle 避障力]
  对前方检测到的障碍物:
    方向 = 障碍法线方向 (沿表面滑行)
    强度 = w_o / distance
    检测方式 = 前方扇形射线检测

[Leader Follow 领头羊跟随力]
  如果视野内有领头羊且自己不是领头羊:
    方向 = (领头羊位置 - 自身位置).normalized
    强度 = w_l (默认 1.5, 强于普通聚合)
```

### 4.2 个体状态机

```lua
--[[
  羊的状态转换图:
  
  [Idle] ←→ [Grazing]
    ↓           ↓
  [Alarmed] ←──┘
    ↓
  [Fleeing]
    ↓ (连续惊吓 ≥3次/10秒)
  [Panicked]
    ↓ (脱离威胁 5秒)
  [Idle]
]]

状态参数:
┌──────────┬────────────┬────────────┬───────────┐
│  状态     │ 移动速度    │ Flee权重    │ 视野半径   │
├──────────┼────────────┼────────────┼───────────┤
│ Idle     │ 0.3 m/s    │ 1.0        │ 6.0 m     │
│ Grazing  │ 0.0 m/s    │ 1.0        │ 4.0 m     │
│ Alarmed  │ 0.5 m/s    │ 2.0        │ 8.0 m     │
│ Fleeing  │ 3.0 m/s    │ 3.0        │ 10.0 m    │
│ Panicked │ 4.0 m/s    │ 0.5 (乱跑) │ 3.0 m     │
└──────────┴────────────┴────────────┴───────────┘

转换条件:
  Idle → Grazing    : 附近有草地且无威胁 (3秒后)
  Grazing → Alarmed : 犬进入感知范围 R_alert (8m)
  Idle → Alarmed    : 犬进入感知范围 R_alert (8m)
  Alarmed → Fleeing : 犬进入逃跑距离 R_flee (5m) 或听到吠叫
  Alarmed → Idle    : 犬离开感知范围 (2秒后)
  Fleeing → Panicked: 10秒内被惊吓 ≥3次
  Fleeing → Alarmed : 犬退出逃跑距离 (1秒后)
  Panicked → Idle   : 脱离所有威胁 (5秒后)
```

### 4.3 领头羊机制

```
领头羊选举算法:
1. 羊群中位置最前方(朝运动方向)的 N 只羊为候选
2. 候选羊中, 速度最高 + 持续领先时间最长的 → 成为领头羊
3. 领头羊影响:
   - 其他羊的 Cohesion 力偏向领头羊 (额外 +50% 权重)
   - 领头羊自身: Flee 权重降低 20% (更"勇敢")
4. 领头羊降级:
   - 被其他羊超过 3秒 → 重新选举
   - 进入 Panicked 状态 → 立即降级
```

### 4.5 性能优化策略

```
[空间分区]
  使用网格(Grid)进行邻居查询
  每只羊只检查自身所在格子 + 相邻8格
  网格大小 = max(R_cohesion, R_flee) ≈ 5m

[更新频率分层]
  ├── Boids 力计算: 每帧 (60fps)
  ├── 状态机转换:   每 5 帧
  ├── 领头羊选举:   每 30 帧
  └── 远处羊群:     可降频到 15fps (距离所有玩家 > 20m)

[数量上限]
  推荐: 单关卡 ≤ 30 只羊 (移动端)
  极限: 50 只 (需启用 LOD 行为简化)
```

## 五、空间地形控制详解

### 5.1 地形对玩法的影响

```
地形本质上是"力场":
┌───────────────────────────────────────────────┐
│                                               │
│   围栏 ═══════╗                               │
│               ║  ← 刚性阻挡 (反弹力)          │
│   草地 · · ·  ║                               │
│         ↑     ║                               │
│       吸引力  ╠══ 栅栏门 (可开关)              │
│    (羊想吃草) ║                               │
│               ║      ~~~~~~~~                 │
│   灌木 ###    ║      ~ 水域 ~  ← 不可通行      │
│     ↑         ║      ~~~~~~~~                 │
│  犬可穿/羊不行 ║                               │
│               ╚═══════════════                │
│                     目标围栏 ◎                 │
└───────────────────────────────────────────────┘
```

### 5.2 关卡设计语言

```
基础元素组合:
  [漏斗]  = 两道平行围栏 + 渐窄通道 → 引导羊群汇聚
  [岔路]  = Y形路径 + 各分支通向不同围栏 → 需要分流
  [瓶颈]  = 窄桥/窄门 → 羊排队通过, 需要耐心
  [死角]  = 三面围栏 → 羊可能被困, 也可利用来集中
  [开阔]  = 无遮挡区域 → 羊容易散开, 需要多人包围
  [绕行]  = 直线被阻挡 → 必须绕路, 增加难度
```

## 六、多人围控协作模型

### 6.1 围控效率公式

```
围控效率 = f(角度覆盖, 距离适当性, 协作一致性)

角度覆盖:
  将羊群重心为中心, 计算各犬所在方向的角度
  理想: N 只犬均匀分布在 360° 上 (但要留出目标方向的缺口)
  实际: 目标反方向(推力侧)至少 1 只犬, 两翼各 1 只

  覆盖评分 = 犬的角度标准差 / 理想角度间隔
  (越接近 1.0 越好)

距离适当性:
  犬距离羊群重心太近 → 恐慌散开
  犬距离羊群重心太远 → 无效果
  最佳距离 ≈ FLEE_RADIUS × 0.8 (≈ 4m)

协作一致性:
  所有犬的"隐含推力方向"是否一致指向目标
  推力方向 = 羊群重心 → 犬位置的反方向
  一致性 = 各犬推力方向与"重心→目标"方向的平均余弦值
```

### 6.2 自然分工涌现

```
不强制分工, 而是通过机制设计使分工自然涌现:

[机制1] 距离最近原则
  掉队的羊 → 最近的犬自然去追
  → 涌现出"收尾者"角色

[机制2] 包围缺口补位
  当一侧无犬时, 羊从该侧逃散
  → 玩家观察到后自然去补位
  → 涌现出"侧翼"角色

[机制3] 吠叫协调
  一人吠叫时其他人应保持位置防止反向逃散
  → 自然形成"一推多挡"节奏

[机制4] 视觉提示
  小地图上高亮无犬覆盖的区域 (红色)
  → 引导玩家主动补位
```

## 七、验证命题分析

### 命题: 2D 牧羊合作游戏 = 羊群 AI 流动 + 空间地形控制 + 多人围控协作

#### 验证1: 羊群 AI 流动 — 乐趣的来源

| 验证点 | 结论 |
|--------|------|
| 是否是核心体验? | **是**。所有参考作品的核心乐趣都来自间接控制产生的涌现行为 |
| Boids 模型是否足够? | **基本足够 + 需要扩展**。需要加入 Flee、状态机、领头羊 |
| 最小可玩单元? | 5只羊 + Boids三力 + Flee力 + 1个围栏目标 |

#### 验证2: 空间地形控制 — 策略的深度

| 验证点 | 结论 |
|--------|------|
| 是否是必要组成? | **是**。无地形约束 = 纯追逐 = 无策略 = 很快枯燥 |
| 地形复杂度下限? | 至少需要: 围栏(阻挡) + 门(可控) + 目标围栏 |
| 地形如何产生策略? | 玩家利用围栏"借力"、利用瓶颈"聚拢"、利用地形"切断"逃跑路线 |

#### 验证3: 多人围控协作 — 体验的放大器

| 验证点 | 结论 |
|--------|------|
| 是否是必要的? | **不是必要但极大增强**。单人可玩, 但多人体验质变 |
| 协作如何产生? | 通过"围控缺口 = 羊逃散"的负反馈自然驱动 |
| 最佳人数? | 2人(最佳体验-复杂度比), 4人(最热闹) |

#### 结论

```
✅ 命题成立, 三要素构成了稳固的三角:

         羊群 AI 流动
        ╱            ╲
      提供             需要
    可操控性           策略空间
      ╱                  ╲
  多人围控协作 ————— 空间地形控制
              相互依赖

- 羊群 AI 是"被操控对象" → 提供不确定性和乐趣
- 空间地形是"规则约束" → 提供策略深度
- 多人协作是"操控方式" → 放大体验和社交乐趣

缺少任一要素:
  无 AI 流动 → 变成普通推箱子
  无空间控制 → 变成无脑追逐
  无多人协作 → 可玩但缺少社交驱动力
```

## 九、附录: 伪代码 — 羊 Agent 核心更新逻辑

```lua
function SheepAgent:Update(dt)
    -- 1. 获取邻居 (空间分区加速)
    local neighbors = spatialGrid:GetNeighbors(self.position, COHESION_RADIUS)
    local threats = spatialGrid:GetThreats(self.position, ALERT_RADIUS)

    -- 2. 状态机更新 (每5帧)
    if frameCount % 5 == 0 then
        self:UpdateState(threats)
    end

    -- 3. 计算各力
    local force = Vector2(0, 0)

    -- Boids 基础力
    force = force + self:CalcCohesion(neighbors)   * W_COHESION
    force = force + self:CalcSeparation(neighbors)  * W_SEPARATION
    force = force + self:CalcAlignment(neighbors)   * W_ALIGNMENT

    -- 逃离力 (根据状态调整权重)
    local fleeWeight = self.stateFleeWeights[self.state] * W_FLEE
    for _, threat in ipairs(threats) do
        local dir = (self.position - threat.position):Normalized()
        local dist = self.position:Distance(threat.position)
        local strength = fleeWeight / math.max(dist, 0.1)

        -- 吠叫加成
        if threat.isBarking then
            strength = strength * BARK_FORCE_MULT
        end

        force = force + dir * strength
    end

    -- 避障力
    force = force + self:CalcObstacleAvoidance() * W_OBSTACLE

    -- 领头羊跟随
    if self.leader and self.leader ~= self then
        local toLeader = (self.leader.position - self.position):Normalized()
        force = force + toLeader * W_LEADER
    end

    -- 4. 限速并应用
    self.velocity = self.velocity + force * dt
    local maxSpeed = self.stateMaxSpeeds[self.state]
    if self.velocity:Length() > maxSpeed then
        self.velocity = self.velocity:Normalized() * maxSpeed
    end

    self.position = self.position + self.velocity * dt

    -- 5. 朝向平滑 (朝速度方向)
    if self.velocity:Length() > 0.1 then
        self.rotation = LerpAngle(self.rotation, self.velocity:Angle(), 5 * dt)
    end
end

function SheepAgent:UpdateState(threats)
    local nearestThreatDist = self:GetNearestThreatDistance(threats)

    if self.state == "idle" then
        if nearestThreatDist < ALERT_RADIUS then
            self:SetState("alarmed")
        elseif self:IsOnGrass() and self.idleTimer > 3.0 then
            self:SetState("grazing")
        end

    elseif self.state == "grazing" then
        if nearestThreatDist < ALERT_RADIUS then
            self:SetState("alarmed")
        end

    elseif self.state == "alarmed" then
        if nearestThreatDist < FLEE_RADIUS or self:HeardBark() then
            self:SetState("fleeing")
        elseif nearestThreatDist > ALERT_RADIUS and self.stateTimer > 2.0 then
            self:SetState("idle")
        end

    elseif self.state == "fleeing" then
        if self.scareCount >= 3 and self.scareTimer < 10.0 then
            self:SetState("panicked")
        elseif nearestThreatDist > FLEE_RADIUS and self.stateTimer > 1.0 then
            self:SetState("alarmed")
        end

    elseif self.state == "panicked" then
        if nearestThreatDist > ALERT_RADIUS and self.stateTimer > 5.0 then
            self:SetState("idle")
        end
    end
end
```
